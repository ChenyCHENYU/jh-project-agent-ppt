# jh-project-agent-ppt 完整改造建议

## 1. 改造目标

这次改造不再以“修单页问题”为目标，而是把整套培训稿从“能展示”提升到“可演讲、可复用、可持续维护”。

目标分成四层：

1. 内容层：每页只保留一个主结论，减少资料堆砌感。
2. 结构层：所有页面信息主次明确，避免标题、说明、例子、结论混在一起。
3. 视觉层：统一风格语言，减少左竖线、散装色块、随手写的内联样式，建立更克制、更高级的质感。
4. 工程层：清理重复和冲突样式，让运行态、源码、构建产物一致，后续修改只需要调少数几个组件。

## 2. 当前项目的核心问题

### 2.1 最大的根问题不是某一页，而是样式系统发生了“并行演化”

在 [styles/index.css](styles/index.css#L461) 到 [styles/index.css](styles/index.css#L523) 这一段，`highlight-box` 和 `section-banner` 已经是新版面板样式。

但在 [styles/index.css](styles/index.css#L1067) 到 [styles/index.css](styles/index.css#L1091) 又被旧版“左边线 + 透明背景”的规则重新覆盖了。

这会直接导致三个问题：

1. 你看到的页面风格不稳定，同一个类名在不同运行态像两个版本。
2. 我们以为“去竖线化”已经完成，实际上后置旧规则又把它改回去了。
3. 后续任何页面继续复用 `highlight-box` / `section-banner`，都会把旧审美继续扩散。

这件事必须列为第一优先级。否则后面继续修页面，等于在不稳定底座上装修。

### 2.2 首页和目录页仍然保留了旧的“信息罗列式首页”思维

虽然首页已经引入了 [slides.md](slides.md#L31) 的 `hero-kicker` 和 [slides.md](slides.md#L65) 的 `metric-stage`，但目录页仍然在 [slides.md](slides.md#L123) 到 [slides.md](slides.md#L179) 使用了大量 `border-l-2` 的卡片。

这会造成两个观感问题：

1. 首页开场页和后续目录页不是同一种设计语言。
2. 目录页像“列表页”，不像“演讲结构导航页”。

目录页现在承担的是整套稿件的结构预期，不应该只是八个带左边线的小卡片。

### 2.3 第 02 章和第 03 章仍然是整套稿件中“资料感最重”的两章

第 02 章虽然第一页全景卡已经提升，但从 [pages/02-l0-l7.md](pages/02-l0-l7.md#L114) 往后，仍有大量：

1. `highlight-box`
2. `layout: two-cols`
3. 大段代码块
4. 解释性段落堆叠

第 03 章问题更集中。从 [pages/03-kit.md](pages/03-kit.md#L29) 开始到 [pages/03-kit.md](pages/03-kit.md#L326)，存在明显的“把文档搬上投影片”的倾向：

1. 代码块过长
2. MCP Tools 页面是信息清单，不是表达页
3. Skills 覆盖图页的层级很多，但视觉焦点不够明确
4. 数据实证页同时承载对比、解释、证明，信息任务过多

这两章是当前整套观感的主要拖累项。

### 2.4 局部页已经变好，但全套语言还没有真正统一

做得比较稳的页面主要集中在：

1. [pages/01-why.md](pages/01-why.md#L22) 到 [pages/01-why.md](pages/01-why.md#L106)
2. [pages/06-workflow.md](pages/06-workflow.md#L22) 到 [pages/06-workflow.md](pages/06-workflow.md#L134)
3. [pages/07-challenges.md](pages/07-challenges.md#L23) 到 [pages/07-challenges.md](pages/07-challenges.md#L160)
4. [pages/08-vision.md](pages/08-vision.md#L23) 到 [pages/08-vision.md](pages/08-vision.md#L201)

这些页面已经比较接近“面板化、结构化、少线条”的目标。

但 [pages/04-ui.md](pages/04-ui.md#L66)、[pages/05-bd.md](pages/05-bd.md#L81)、[pages/02-l0-l7.md](pages/02-l0-l7.md#L178) 这类页面，仍然混用了两种表达体系：

1. 一部分是新面板体系
2. 一部分仍然是旧式说明块 + 大代码块 + two-cols 模板

结果就是“单页看还行，整套翻过去不够高级”。

### 2.5 当前稿件更像“培训资料”，还不够像“高质量演讲稿”

主要表现为：

1. 单页结论不够尖锐
2. 说明文字偏多，视觉停顿偏少
3. 一些页在试图同时完成“定义、举例、证明、结论”四件事
4. 页面之间缺少节奏变化

换句话说，现在的主要矛盾已经不是“页会不会溢出”，而是“讲的时候有没有压迫感、记忆点和说服力”。

## 3. 改造原则

### 3.1 先统一底层样式，再修页面

第一步不是继续逐页精修，而是先把样式体系收口，不再允许同一组件多处定义、后置覆盖。

### 3.2 每页只保留一个主结论

一页里只能有一个主标题意图，例如：

1. 提出问题
2. 解释机制
3. 展示结构
4. 给出证据
5. 做出判断

如果一页同时承担两种以上任务，就拆或减。

### 3.3 用“面”表达层级，不再用“线”硬切层级

后续视觉语言应以：

1. 面板
2. 轻渐变
3. 胶囊标签
4. 数字主视觉
5. 小体量辅助点缀

作为主要语言。

左边线、硬边框、强分隔符只作为少量强调，不再作为默认结构语法。

### 3.4 代码块和文档块不直接搬上 PPT

投影片不是文档页。后续处理原则：

1. 长代码块只保留结构片段
2. 长目录结构只展示关键层
3. 工具清单只保留分组和代表项
4. 其余信息放进讲稿备注或补充页，不占主线

### 3.5 统一页面节奏

全套页面应该形成稳定节奏：

1. 章节页：情绪起势
2. 结构页：建立框架
3. 证据页：给出事实或案例
4. 方法页：给出路径
5. 愿景页：抬高视角

这样整套稿件才像完整叙事，不是页面拼盘。

## 4. 建议的统一视觉体系

### 4.1 保留的组件

这批组件可以作为正式基线继续用：

1. [styles/index.css](styles/index.css#L118) 的 `panel-soft-*`
2. [styles/index.css](styles/index.css#L165) 的 `accent-top-card*`
3. [styles/index.css](styles/index.css#L212) 的 `layer-row`
4. [styles/index.css](styles/index.css#L510) 的 `hero-kicker`
5. [styles/index.css](styles/index.css#L532) 的 `metric-stage`
6. [styles/index.css](styles/index.css#L589) 的 `capability-card*`

这些组件已经具备统一系统的雏形。

### 4.2 应该降级或淘汰的表达方式

建议不再把下面这些作为主流写法继续扩散：

1. `highlight-box` 系列旧版左边线表达
2. `section-banner` 的竖线式标题条
3. 大量 `border-l-2` 卡片
4. 大段内联 `style="background... border..."`
5. 过长的代码块直接上屏
6. `layout: two-cols` 下靠 `pt-8 / pr-4 / pl-4` 微调的布局方式

### 4.3 建议新增的组件语法

为了把后续页面改得更统一，建议新增四类组件：

1. `statement-card`：用于“这一页的结论”
2. `evidence-strip`：用于横向展示 2 到 4 个关键证据
3. `compare-panel`：统一所有 before / after、传统 / 新方案对比页
4. `slide-eyebrow + slide-summary`：统一每页顶部引导和底部一句话结论

这样后续页就不需要靠临时拼装。

## 6. 实施顺序建议

建议分三轮做，而不是一次性全改。

### 第一轮：底层收口

只做系统，不做大面积逐页美化。

范围：

1. 清理 [styles/index.css](styles/index.css#L821) 与 [styles/index.css](styles/index.css#L1067) 这类重复覆盖
2. 固化 `highlight-box` / `section-banner` / 目录卡片的最终样式语法
3. 建立统一的 compare-panel、statement-card 等组件
4. 替换首页目录页的旧边线表达

预期结果：视觉语言统一，不再反复被旧规则打回去。

### 第二轮：重点重构第 02 章和第 03 章

范围：

1. [pages/02-l0-l7.md](pages/02-l0-l7.md)
2. [pages/03-kit.md](pages/03-kit.md)

原因：

1. 它们最影响整套观感
2. 它们最像资料，不像演讲稿
3. 它们一旦收好，整套稿件气质会明显上一个台阶

预期结果：把最弱两章拉到统一基线。

### 第三轮：全稿精修与节奏统一

范围：

1. 首页与目录页
2. 第 04、05、07、08 章细修
3. 页间语气、结论句、收尾句统一

预期结果：整套稿件从“页面正确”进入“可以拿出去讲”的状态。

## 7. 验收标准

这次改造建议按下面标准验收，而不是只看“有没有溢出”。

1. 不再出现大面积左竖线作为默认设计语言。

### 7.2 内容验收

2. 页面不再像文档摘录，而是演讲表达。
3. 第 02 章和第 03 章明显摆脱资料感。

### 7.3 工程验收

1. [styles/index.css](styles/index.css) 不再出现同类组件多处互相覆盖。
2. `slidev build` 持续通过。
3. 全页无溢出。
4. 运行态和构建态样式一致，不再出现“源码是新版、页面却像旧版”的情况。
