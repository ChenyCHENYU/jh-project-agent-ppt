# 从设计令牌到运行时渲染：UI 的 5 层进阶

<div class="mt-3 space-y-2">

<div class="layer-row">
  <div class="level-tag l0 flex items-center justify-center px-3 py-1.5 text-sm shrink-0 w-12">L0</div>
  <div class="layer-body layer-body-blue">
    <div class="layer-title"><div class="layer-title-main"><span class="font-semibold text-blue-300 text-sm">Design Tokens · 设计令牌</span><span class="layer-path">design/tokens/base.css</span></div><span class="layer-kicker-inline">Foundation <span class="layer-kicker-cn">基础层</span></span></div>
    <div class="layer-summary">颜色 / 间距 / 圆角 / 字号 / 阴影，所有上层依赖的统一设计源。</div>
    <div class="layer-note">改一处，全项目视觉基线同步生效。</div>
  </div>
</div>

<div class="layer-row">
  <div class="level-tag l1 flex items-center justify-center px-3 py-1.5 text-sm shrink-0 w-12">L1</div>
  <div class="layer-body layer-body-indigo">
    <div class="layer-title"><div class="layer-title-main"><span class="font-semibold text-indigo-300 text-sm">Element Plus 原子层</span><span class="layer-path">styles/element/</span></div><span class="layer-kicker-inline">Atomic <span class="layer-kicker-cn">原子层</span></span></div>
    <div class="layer-summary">el-button / el-input / el-table 等基础控件逐一校准。</div>
    <div class="layer-note">一控件一规则，收敛差异而不牵连其他控件。</div>
  </div>
</div>

<div class="layer-row">
  <div class="level-tag l2 flex items-center justify-center px-3 py-1.5 text-sm shrink-0 w-12">L2</div>
  <div class="layer-body layer-body-purple">
    <div class="layer-title"><div class="layer-title-main"><span class="font-semibold text-purple-300 text-sm">Vendors 封装组件化妆层</span><span class="layer-path">styles/vendors/</span></div><span class="layer-kicker-inline">Vendor Skin <span class="layer-kicker-cn">封装层</span></span></div>
    <div class="layer-summary">Base* / jh-* / C_*/c_* / AG Grid 的统一化妆入口。</div>
    <div class="layer-note">老项目主战场，无源码也能覆盖视觉与控件体验。</div>
  </div>
</div>

<div class="layer-row">
  <div class="level-tag l3 flex items-center justify-center px-3 py-1.5 text-sm shrink-0 w-12">L3</div>
  <div class="layer-body layer-body-cyan">
    <div class="layer-title"><div class="layer-title-main"><span class="font-semibold text-cyan-300 text-sm">Page Layouts · 骨架层</span><span class="layer-path">styles/layouts/</span></div><span class="layer-kicker-inline">Page DNA <span class="layer-kicker-cn">骨架层</span></span></div>
    <div class="layer-summary">list-page / tree-list / form-dialog 等团队统一页面骨架。</div>
    <div class="layer-note">让页面结构先统一，再谈业务差异和视觉表达。</div>
  </div>
</div>

<div class="layer-row">
  <div class="level-tag l4 flex items-center justify-center px-3 py-1.5 text-sm shrink-0 w-12">L4</div>
  <div class="layer-body layer-body-green">
    <div class="layer-title"><div class="layer-title-main"><span class="font-semibold text-green-300 text-sm">Runtime 业务渲染层</span><span class="layer-path">runtime/</span></div><span class="layer-kicker-inline">Execution <span class="layer-kicker-cn">执行层</span></span></div>
    <div class="layer-summary">defineColumns / renderOps / preset，把业务页面变成声明式配置。</div>
    <div class="layer-note">让业务代码尽量“消失”，只在有源码控制时启用。</div>
  </div>
</div>

</div>

# L4 Runtime：让业务代码"消失"

<div class="compare-stage mt-4 text-sm">

<div class="compare-panel compare-panel-before">
  <div class="compare-heading">
    <div class="font-bold text-slate-100">没有 Runtime</div>
    <span class="badge badge-wip">重复配置</span>
  </div>

  <div class="statement-card text-sm">
    <strong>问题：</strong>表格列定义、状态渲染、操作列拼装都要在每个页面里重复手写。
  </div>

  <div class="arch-card text-[11px] leading-5 font-mono">
    <div>const columns = [</div>
    <div>&nbsp;&nbsp;{ field: 'status', cellRenderer: (...) =&gt; tag }</div>
    <div>&nbsp;&nbsp;{ field: 'createTime', formatter: (...) }</div>
    <div>&nbsp;&nbsp;{ field: 'amount', formatter: (...) }</div>
    <div>]</div>
    <div class="text-slate-500 mt-2">同类模板在每个表格里重复出现。</div>
  </div>
</div>

<div class="compare-panel compare-panel-after">
  <div class="compare-heading">
    <div class="font-bold text-slate-100">有 Runtime</div>
    <span class="badge badge-done">声明式配置</span>
  </div>

  <div class="statement-card statement-card-green text-sm">
    <strong>结果：</strong>业务代码只声明字段和意图，渲染逻辑交给 Runtime 自动完成。
  </div>

  <div class="compare-code-stack text-[11px] leading-5 font-mono">
    <div class="arch-card-green">defineColumns([{ field: 'status', renderType: 'tag', dictCode: 'COMMON_STATUS' }])</div>
    <div class="arch-card-cyan">renderOps([{ label: '编辑', type: 'primary' }, { label: '删除', confirm: true }])</div>
  </div>

  <div class="statement-card text-sm">
    <strong>COLUMN_AUTO_MAP：</strong>字段名含 <code>time/date</code> → datetime，含 <code>amount/price</code> → money，含 <code>status</code> → tag。
  </div>
</div>

</div>
