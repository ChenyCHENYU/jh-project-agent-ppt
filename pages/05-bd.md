# api.md — 唯一交汇点

<div class="compare-stage mt-4 text-sm">

<div class="compare-panel compare-panel-after">
  <div class="statement-card statement-card-green">
    <strong>不是为了存档，而是为了让双端对齐</strong><br>
    前端生成它，后端消费它，AI 两边都读它。它的价值不是“写了一份文档”，而是把双方实现拉回同一份契约。
  </div>

  <div class="panel-soft text-sm">
    <div class="grid grid-cols-3 gap-3 text-center">
      <div>
        <div class="text-cyan-400 font-bold text-base mb-1">前端</div>
        <div class="text-slate-500 text-xs">api-contract Skill<br>→ 生成 api.md</div>
      </div>
      <div class="flex items-center justify-center text-slate-600 text-2xl font-thin">⇄</div>
      <div>
        <div class="text-green-400 font-bold text-base mb-1">后端</div>
        <div class="text-slate-500 text-xs">service-codegen<br>← 消费 api.md</div>
      </div>
    </div>
  </div>

  <div class="compare-panel compare-panel-before !p-4 text-xs">
    <div class="text-red-300 font-bold mb-2">质量门控</div>
    <div class="space-y-1.5 text-slate-400">
      <div>☑ 前端：api.md 存在 → 才能进菜单同步</div>
      <div>☑ 后端：api.md 存在 → 才能执行 service-codegen</div>
      <div>☑ 联调：双端不对 → 改 api.md，而不是改代码</div>
    </div>
  </div>
</div>

<div class="compare-panel">
  <div class="arch-card text-[11px] leading-5 font-mono">
    <div># 产品管理 API 契约</div>
    <div class="mt-2">GET /api/product/page</div>
    <div>功能：分页查询产品列表</div>
    <div>Skill：由 api-contract 生成</div>
    <div>后端：Controller → ProductController.page()</div>
    <div class="mt-2 text-slate-500">质量门控</div>
    <div>- api.md 存在，才能进同步</div>
    <div>- api.md 存在，才能执行业务骨架生成</div>
    <div>- 双端不一致，先改契约</div>
  </div>

  <div class="statement-card text-sm">
    <strong>为什么它是唯一交汇点？</strong><br>
    因为一旦前后端出现偏差，修正点应该回到契约本身，而不是各自在代码里打补丁。AI 能稳定消费的，从来不是口头共识，而是机器可读的统一约束。
  </div>
