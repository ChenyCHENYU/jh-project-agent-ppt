<h1 class="tight-title">L0：氛围编程 <span class="tight-title-sub">让 AI 感知你的项目</span></h1>

<div class="page-fit grid-tight-2 text-[13px] leading-5">

<div class="flex flex-col gap-2.5">

<div class="highlight-box">
<strong>核心思想</strong><br>
<span class="text-slate-300">"氛围编程"（Ambient Programming）指的是：</span><strong class="text-blue-300">主动把项目上下文写成文件，放在 AI 能读到的位置</strong>，让 AI 在每次对话时都自动感知项目规范——无需每次手动贴。
</div>

<div class="arch-card space-y-1.5">
<div class="text-slate-400 mb-1">注入哪些文件</div>
<ul class="space-y-0.5 text-slate-300">
  <li>📁 <code>.github/copilot-instructions.md</code> — Copilot 主规范入口</li>
  <li>📁 <code>.cursor/rules/</code> — Cursor 规则文件</li>
  <li>📁 <code>.windsurfrules</code> — Windsurf 规则文件</li>
  <li>📁 <code>CLAUDE.md</code> — Claude Code 规范</li>
  <li>📁 <code>AGENTS.md</code> — 通用 Agent 规范</li>
</ul>
<div class="text-xs text-green-400 mt-1">✅ wl-skills-kit 自动生成以上所有文件</div>
</div>

<div class="arch-card-green">
<div class="font-bold mb-1">一行命令完成 L0 配置</div>
<code class="text-green-300">npx @agile-team/wl-skills-kit</code>
<div class="text-slate-400 mt-1">执行后，10 种编辑器规则文件 + 10 个 SKILL.md 一次性写入项目根目录</div>
</div>

</div>

<div class="flex flex-col gap-2.5">

<div class="grid grid-cols-2 gap-2">
<div class="arch-card-orange text-center py-2.5">
  <div class="text-xs text-slate-400 mb-1">❌ 无 L0 注入</div>
  <div class="text-orange-300 font-bold text-sm">通用 AI</div>
  <div class="text-xs text-slate-400 mt-1">输出通用 el-table<br>字段名自行猜测<br>忽略团队规范</div>
</div>

</div>

<div class="arch-card code-panel text-slate-300">

# L1：提示词工程

<div class="compare-stage mt-4 text-sm">

<div class="compare-panel compare-panel-neutral">
  <div class="statement-card statement-card-indigo">
    <strong>比 L0 更进一步</strong><br>
    L0 解决的是“让 AI 看见文件”，L1 解决的是“让 AI 在正确时机看见正确文件”。
  </div>

  <div class="feature-list">
    <div class="feature-item">
      <div class="feature-mark">1</div>
      <div><strong class="text-indigo-300">懒加载规范</strong><br><span class="text-slate-400">按任务触发规范，不把所有上下文一次性塞满。</span></div>
    </div>
    <div class="feature-item">
      <div class="feature-mark">2</div>
      <div><strong class="text-indigo-300">意图路由</strong><br><span class="text-slate-400">同一句话进入不同流程，核心是识别用户真正要完成的任务。</span></div>
    </div>
    <div class="feature-item">
      <div class="feature-mark">3</div>
      <div><strong class="text-indigo-300">质量门控语言</strong><br><span class="text-slate-400">把“必须 / 禁止 / 验证点”写清楚，让 AI 知道边界。</span></div>
    </div>
  </div>
</div>

<div class="compare-panel">
  <div class="compare-heading">
    <div class="font-bold text-slate-100">规范不是一整坨文档，而是可路由的上下文</div>
    <span class="badge badge-new">L1</span>
  </div>

  <div class="arch-card text-[11px] leading-5 font-mono">
    <div class="text-slate-500 mb-2">Intent Router</div>
    <div>"做个页面 / 出页面" → <span class="text-cyan-300">page-codegen</span></div>
    <div>"扫原型" → <span class="text-cyan-300">prototype-scan</span></div>
    <div>"菜单同步" → <span class="text-cyan-300">menu-sync</span></div>
    <div>"风格不对 / 样式修" → <span class="text-cyan-300">doctor-ui</span></div>
  </div>

  <div class="arch-card-cyan text-[11px] leading-5 font-mono">
    <div class="text-slate-500 mb-2">Standards Lazy Load</div>
    <div>生成页面前 → 加载 <span class="text-cyan-300">page-codegen/SKILL.md</span></div>
    <div>操作表格 → 加载 <span class="text-cyan-300">standards/base-table.md</span></div>
    <div>字典枚举 → 加载 <span class="text-cyan-300">standards/dict-usage.md</span></div>
  </div>

  <div class="statement-card statement-card-green text-sm">
    <strong>关键设计：</strong>上下文是有限资源。结构化文件 + 按需加载，比大段自然语言说明稳定得多。
  </div>
</div>

</div>

<h1 class="tight-title">L2-L3：Skill 固化经验，MCP 引入事实</h1>

<div class="page-fit compare-stage compare-stage-tight text-[13px] leading-5">

<div class="compare-panel compare-panel-neutral">
  <div class="statement-card statement-card-indigo">
    <strong>L2：Skill = 可复用的 AI 任务单元</strong><br>
    一份 <code>SKILL.md</code>，把输入、步骤、输出、质量要求固定成一个可重复调用的任务模板。
  </div>

  <div class="feature-list">
    <div class="feature-item">
      <div class="feature-mark">入</div>
      <div><strong class="text-purple-300">输入格式</strong><br><span class="text-slate-400">原型、截图、口述需求，都可以统一进入同一任务。</span></div>
    </div>
    <div class="feature-item">
      <div class="feature-mark">步</div>
      <div><strong class="text-purple-300">处理步骤</strong><br><span class="text-slate-400">扫描页面层级、提取字段交互、整理结构化清单。</span></div>
    </div>
    <div class="feature-item">
      <div class="feature-mark">出</div>
      <div><strong class="text-purple-300">固定产物</strong><br><span class="text-slate-400">输出报告、字段表、备注说明，供下一步继续消费。</span></div>
    </div>
  </div>

  <div class="arch-card text-[10.5px] leading-4.5 font-mono">
    <div class="text-slate-500 mb-2">prototype-scan / 典型输出</div>
    <div>Input → 原型文件 / 截图 / 口述</div>
    <div>Steps → 扫描层级 / 提取字段 / 识别状态</div>
    <div>Output → pages-list / 特殊逻辑备注 / 结构化清单</div>
  </div>
</div>

<div class="compare-panel">
  <div class="statement-card statement-card-cyan">
    <strong>L3：MCP = 突破上下文限制</strong><br>
    AI 不再只靠静态文件猜，而是可以在对话中实时查询菜单、字典、权限和项目结构。
  </div>

  <div class="info-matrix info-matrix-2 text-[12px] leading-5">
    <div class="compare-panel compare-panel-before !p-4">
      <div class="font-bold text-red-300">没有 MCP</div>
      <div class="text-slate-400">“菜单有哪些模块？” → 不知道或猜错</div>
      <div class="text-slate-400">“字典值有哪些？” → 只能按经验瞎猜</div>
    </div>
    <div class="compare-panel compare-panel-after !p-4">
      <div class="font-bold text-green-300">有了 MCP</div>
      <div class="text-slate-400">调用 <code>wls_get_menu_list</code> → 实时查后台</div>
      <div class="text-slate-400">调用 <code>wls_get_dict_values</code> → 实时取字典</div>
    </div>
  </div>

  <div class="statement-card statement-card-green text-[12px] leading-5">
    <strong>17 个 MCP Tools</strong> 覆盖菜单管理、字典权限、项目感知、页面校验、通知推送五类关键场景。
  </div>
</div>

</div>

# L4–L5：从工具接入，到整链跑通

<div class="compare-stage mt-4 text-sm">

<div class="compare-panel compare-panel-neutral" style="font-size:0.9em">
  <div class="statement-card statement-card-indigo">
    <strong>L4：CLI = 工程化脚手架</strong><br>
    把初始化、升级、校验、导出这些重复动作收敛成命令入口，降低团队接入门槛。
  </div>

  <div class="feature-list text-sm font-mono">
    <div class="feature-item"><div class="feature-mark">$</div><div><span class="text-slate-200">npx @agile-team/wl-skills-kit</span><br><span class="text-slate-500">init 安装</span></div></div>
    <div class="feature-item"><div class="feature-mark">$</div><div><span class="text-slate-200">wl-skills-kit update / diff / validate</span><br><span class="text-slate-500">升级、差异查看、体检</span></div></div>
    <div class="feature-item"><div class="feature-mark">$</div><div><span class="text-slate-200">npm run standards:init</span><br><span class="text-slate-500">规范插件初始化</span></div></div>
    <div class="feature-item"><div class="feature-mark">$</div><div><span class="text-slate-200">npm run version:verify / lint:skills</span><br><span class="text-slate-500">版本一致性与 Skill 静态检查</span></div></div>
  </div>
</div>

<div class="compare-panel">
  <div class="statement-card statement-card-cyan">
    <strong>L5：Agent Pipeline = 多步自动化流水线</strong><br>
    多个 Skill 按顺序协同推进，人只在真正高风险的节点介入确认，目标不是演示单点能力，而是真实页面连续跑通。
  </div>

  <div class="arch-card text-[11px] leading-5 font-mono">
    <div>需求输入</div>
    <div>↓ prototype-scan / business-doc</div>
    <div>↓ api-contract / page-codegen</div>
    <div>↓ validate-page / convention-audit / code-fix</div>
    <div>↓ menu / dict / permission-sync</div>
    <div class="text-amber-300">→ 仅同步前需要人工确认</div>
  </div>

  <div class="statement-card statement-card-green text-sm">
    <strong>当前状态：</strong> L5 协议已落地，下一步不是再加概念，而是把整条链路在真实页面上稳定连续跑通。
  </div>
</div>

</div>
