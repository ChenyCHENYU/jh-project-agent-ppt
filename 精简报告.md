## 三、精简四原则

1. **聚焦标准化**：只留"如何把开发阶段标准化"的主线，砍愿景/Hermes/自演化等"未来篇"。
2. **删概念、留实证**：L0–L7 八层详解是内部框架词汇，对外听众无感 → 压成 1 页全景图。
3. **四包为主线**：design / kit / ui / bd 各占其位，用 `api.md` 契约串成一条线。
4. **实战与数据为王**：演示页、量化效果（30→85、0 偏差、返工骤降）是 20 分钟里最有说服力的部分，全部保留并强化。
